## 一、产品概述

### 1.1 背景与目标

清流是一款帮助用户建立健康习惯的数字工具，面向希望减少或戒断色情/手淫成瘾行为的人群。产品以神经科学原理为根基，结合认知行为疗法（CBT）和接纳承诺疗法（ACT），为用户提供科学的渴望管理支持。

### 1.2 核心价值主张

> "用科学的方式，陪你在渴望的每一刻做出不同的选择。"

我们不是通过羞耻驱使用户，而是通过神经科学验证的即时干预技术，帮助用户在渴望高峰时（通常持续90秒左右）打断自动化行为回路，逐步重建前额叶对习惯系统的抑制控制。

### 1.3 目标用户

- 18-35岁男性为主，有成瘾困扰，希望改变但缺乏支持工具
- 有一定自我觉察，认同科学方法，不接受伪科学叙事
- 需要即时的、3秒内可操作的渴望应对支持

## 二、理论基础

### 2.1 三维干预框架

产品的所有功能设计均基于以下三维干预模型：

| 维度 | 核心机制 | 产品对应功能 |
|------|---------|-------------|
| **神经层面** | VTA-NAc多巴胺奖赏通路 + 习惯/目标导向双重系统 | 戒断计时器（神经恢复可视化） |
| **认知层面** | CBT/ACT + Marlatt复发预防模型 | 渴望应对面板（CBT练习）+ 触发日志 |
| **情感层面** | 羞耻-复发循环解耦 + 自我效能感培养 | 破戒后支持 + 进度报告（去羞耻化叙事） |

### 2.2 关键科学结论（来源：研究文件）

**神经科学核心发现**：
- 渴望高峰平均持续约90秒，之后自然衰减（神经递质耗竭机制）
- 前额叶抑制控制减弱是成瘾核心问题，打断渴求需增强前额叶功能
- 多巴胺受体恢复需要90天以上，神经可塑性改变需要持续干预

**心理学核心发现**：
- CBT/ACT对问题性成瘾行为产生大效应量（SMD≈1.05），是本领域唯一有强Meta分析支持的核心干预方法
- 羞耻感通过激活背侧纹状体形成负性强化循环，是复发的重要中介变量（β=0.42）
- 破戒≠失败（Marlatt模型），将堕落重构为学习机会是预防复发的关键

**行为科学核心发现**：
- 3秒内响应能力是用户体验的核心差异点
- 每日双端结构（早晨承诺+夜间回顾）有助于建立持续的行为干预习惯
- 社会联结使成瘾易感性个体风险降低52%

### 2.3 不使用超能力叙事

产品不使用"重启"、"超能力"等伪科学叙事。所有用户可见的描述均基于可验证的神经科学结论，如"神经恢复进程"、"多巴胺自然衰减"等。

### 3.2 戒断计时器

**功能描述**：显示用户连续戒断时长，配套里程碑庆祝和神经恢复可视化。

#### 3.2.1 核心逻辑

- 计时精度：秒级
- 数据结构：记录开始时间、破戒时间戳（用于破戒后分析），不丢失历史数据
- 失败后处理：破戒后保留累计天数和已建立的技能/学习记录，仅清零当前连续天数

#### 3.2.2 里程碑设计

| 里程碑 | 时间 | 庆祝方式 | 神经科学依据 |
|--------|------|---------|------------|
| 1天 | 24小时 | 基础庆祝动画 | 急性期：多巴胺波动开始稳定 |
| 7天 | 168小时 | 中级庆祝动画 | 早期：习惯系统开始重校准 |
| 14天 | 新增 | 庆祝动画 | 中期：自我效能感建立窗口期 |
| 30天 | 720小时 | 高级庆祝动画 + 成就徽章 | 神经递质水平初步恢复 |
| 60天 | 新增 | 庆祝动画 | 中长期：前额叶功能持续改善 |
| 90天 | 2160小时 | 特殊庆祝动画 + 报告卡片 | 多巴胺受体基本恢复（文献支持但有不确定性） |

#### 3.2.3 神经恢复可视化

用科学语言描述各阶段变化，不使用"重生"、"超能力"等表述：

- 第1周："多巴胺受体开始适应非人工刺激的节奏"
- 第2周："大脑的注意力资源正在重新分配"
- 第30天："神经递质水平正在逐步恢复基线"
- 第90天："神经系统的恢复进程持续进行（多巴胺受体恢复需要约90天）"

### 3.3 渴望应对面板（P0，需重点升级）

**功能描述**：用户在渴望来袭时，可<3秒打开的即时响应面板，提供科学验证的干预技术。

#### 3.3.1 优先级排序（基于tech_mechanism.md研究）

| 优先级 | 技术 | 核心机制 | 预计开发 |
|--------|------|---------|---------|
| **最优先** | T11：冷水洗脸+自我对话 | 冷激活+认知解离双重协同 | 1周 |
| **高优先** | T4：90秒呼吸法 | 呼气延长→副交感神经激活 | 1周 |
| **高优先** | T5：感官接地54321 | 前额叶招募→杏仁核抑制 | 1周 |
| 中优先 | T2：爆发性运动 | 前额叶激活→抑制性控制增强 | 可选 |
| 中优先 | T1：冷水激活（面部） | 潜水反射→副交感激活 | 可选 |

> 注：T8屏息冲刺、T10强震感换场证据强度较弱，暂不作为核心推荐。

#### 3.3.2 技术实现规格

**T4 90秒呼吸法（最高优先级）**

```
步骤：
1. 吸气4秒
2. 屏息2秒
3. 缓慢呼气15秒
4. 重复6轮（共约90秒）

UI要求：
- 提供视觉呼吸引导动画（气泡/圆圈呼吸动画）
- 可选：显示"心率下降"提示
- 操作路径：从渴望面板<3秒可达
- 结束时提示："渴望高峰通常持续90秒，你刚刚度过了一个完整周期"
```

**T5 感官接地54321（高优先级）**

```
步骤：
1. 说出5件你能看见的东西
2. 说出4件你能触摸的东西
3. 说出3件你能听见的声音
4. 说出2件你能闻到的气味
5. 说出1件你能尝到的味道

UI要求：
- 分步骤引导界面（每步一个页面）
- 可选：语音辅助模式
- 操作路径：从渴望面板<3秒可达
```

**T11 冷水洗脸+自我对话（最优先，完整方案）**

```
步骤：
阶段1（冷水激活，30-60秒）：
1. 冷水敷面
2. 感受温度激活的清醒感

阶段2（认知对话，接续冷水）：
1. 对着镜子说（或默念）：
   "这只是神经习惯的回响，不是我的本质"
   "90秒后这个渴望会自然衰减"
   "我选择给前额叶一个机会"

UI要求：
- 分两阶段引导（先物理干预后认知对话）
- 提供标准化自我对话模板
- 可选：镜子模式的视觉引导
```

#### 3.3.3 面板入口设计

- 主屏幕快捷入口（图标+文字）
- App启动时快速入口
- 推送通知触发："渴望来袭？3秒打开清流"
- 从任何页面均可<3秒触达

### 3.4 触发日志

**功能描述**：记录触发场景、情绪状态、应对结果，支持AI辅助的模式识别。

#### 3.4.1 记录字段

| 字段 | 类型 | 说明 |
|------|------|------|
| 时间戳 | 自动 | 记录时间 |
| 触发场景 | 选择/自由文本 | 地点、时间、前置事件 |
| 情绪状态 | 多选标签 | 焦虑/抑郁/孤独/无聊/愤怒/压力/其他 |
| 渴望强度 | 1-10量表 | 用户自评 |
| 应对方式 | 多选 | 使用了哪种技术/未应对/其他 |
| 结果 | 选择 | 成功抵抗/破戒/部分成功 |
| 备注 | 自由文本 | 其他想记录的内容 |

#### 3.4.2 情绪-触发关联分析（P1升级项）

- 情绪热力图：展示哪种情绪状态下最常出现触发
- 高风险情境清单：基于用户历史数据生成个人化风险情境
- 当用户记录负面情绪时，主动推送渴望应对提示

### 3.5 进度报告

**功能描述**：展示用户的戒断进展、应对记录、正向数据。

#### 3.5.1 数据展示

- 连续戒断天数（含历史最佳）
- 本周渴望应对次数（正向统计："本周你成功应对了X次渴望"）
- 里程碑达成情况
- 睡眠/情绪趋势（周视图）

#### 3.5.2 去羞耻化叙事原则

| ❌ 避免 | ✅ 使用 |
|--------|--------|
| "失败了，从头开始" | "你已经建立了X天的恢复基础，这是宝贵的" |
| "连续天数重置为0" | "你的技能和联结仍在，继续前进" |
| "完美戒断" | "持续进步，而非完美" |
| 大字显示"你失败了" | 破戒页面以"发生了什么？"引导，而非"你失败了" |

### 3.6 破戒后支持系统（P0新增）

**功能描述**：重新设计破戒后的交互流程，从"羞耻驱动"转变为"学习驱动"。

#### 3.6.1 破戒流程设计

```
步骤1：破戒识别
- 检测到用户记录破戒时，不显示默认重置页面
- 显示："让我们理解这次发生了什么"

步骤2：破戒分析问卷（3-5个问题，<2分钟完成）
- "这次破戒发生在什么情境下？"
- "破戒前你的情绪状态是什么？"
- "破戒前的渴望强度是多少？"
- "这次破戒中你学到了什么？"
- "下一次遇到类似情境，你会怎么做不同？"

步骤3：科学叙事呈现
显示："这是神经习惯的回响，不是你的失败。
你的前额叶正在学习一个新的反应方式，这需要时间和重复。"

步骤4：进度保留提示
显示："你仍保有[X]天的累计恢复时间，
和[X]次成功应对的经验。"

步骤5：下一步微行动
推荐1-2个具体可执行的微步骤（如："做一次90秒呼吸法"）
```

#### 3.6.2 数据保留规则

| 数据类型 | 破戒后处理 |
|---------|-----------|
| 累计戒断天数 | 保留（不清零） |
| 已学到的触发情境 | 保留 |
| 成功应对记录 | 保留 |
| 技能徽章/成就 | 保留 |
| 当前连续天数 | 清零 |
| 里程碑进度 | 清零（重新计算） |

### 3.7 每日双端结构干预（P1新增）

**功能描述**：参考行业最佳实践（I Am Sober），设计每日干预框架。

#### 3.7.1 早晨承诺（打开App <10秒完成）

- 显示当日戒断宣言（一句话）
- 可选：设定当日具体目标（如"今天如有渴望，我会用T4呼吸法"）
- 简短数据回顾（昨日表现、正向进展）

#### 3.7.2 夜间回顾（睡前2-3分钟）

- "今天我应对了哪些挑战？"（促进元认知）
- "今天我的状态如何？"（情绪自评1-5）
- "今天我为恢复做了什么积极的事？"（强化正向行为）
- 简短统计（连续天数、本周应对次数）

#### 3.7.3 推送通知

- 早晨提醒："新的一天开始了，今天你有什么承诺？"
- 夜间提醒："睡前花2分钟回顾今天"
- 渴望高风险时间推送（根据用户历史数据学习）

## 四、伦理边界（禁止事项）

### 4.1 绝对禁止

| 禁止内容 | 原因 |
|---------|------|
| 任何形式的公开羞辱或公开破戒记录 | 羞耻驱动会激活负性强化循环，加重成瘾 |
| 将破戒描述为"道德失败"或"意志力不足" | 不符合成瘾神经科学事实，损害用户自我效能 |
| 过度游戏化（装备系统、升级、虚拟货币） | 成瘾用户对奖励系统敏感，可能适得其反 |
| 伪科学叙事（"超能力"、"重启"、"觉醒"） | 长期损害产品可信度，违反产品定位 |
| 自动化预测性干预（预测用户即将破戒并主动干预） | 当前AI准确度不足，误报损害信任 |

### 4.2 慎重对待

| 谨慎使用 | 说明 |
|---------|------|
| 公共排行榜或公开戒断天数 | 可能引发表演性康复，需默认关闭，选择性公开 |
| 财务承诺机制（如押金押注） | 对脆弱用户可能造成伤害，需明确的退出机制 |
| 社群功能 | 需严格的内容审核机制，防止毒性内容 |

## 八、风险提示

1. **引用不确定性**：部分神经科学引用尚无法完全核实，90天恢复时间线从物质成瘾研究外推，色情成瘾特异性数据有限。
2. **伦理合规**：涉及敏感人群，需确保数据隐私和心理安全，避免任何可能强化羞耻的内容。
3. **临床边界**：产品是支持工具而非临床治疗，需明确告知用户不替代专业心理咨询或治疗。
4. **AI伦理**：AI对话功能需经过临床专家审核，防止有害建议。
