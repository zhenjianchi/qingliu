## 二、MVP Feature清单

### Feature 01：戒断计时器增强版

**产品经理描述**

用户需要一个准确、可信、有温度的戒断计时器。它不只是显示天数，而是用神经科学的语言告诉用户"你的大脑正在恢复什么"，让用户在每次打开App时感受到进步而不是焦虑。

**功能描述**

1. **秒级精确计时**：记录开始时间、破戒时间戳，支持历史数据保留
2. **里程碑庆祝体系**：1/7/14/30/60/90天，配套动画和成就徽章
3. **神经恢复可视化**：用科学语言描述各阶段进展（替代"超能力"叙事）
4. **破戒数据保留**：破戒后保留累计天数和技能记录，仅清零当前streak

**验收标准（AC）**

- [ ] 计时器精度秒级，跨日期/跨App重启数据不丢失
- [ ] 破戒后"累计天数"不清零，仅"当前连续天数"清零
- [ ] 里程碑页面使用神经恢复科学叙事，无"重启"/"超能力"字样
- [ ] 里程碑庆祝动画<3秒可跳过，不强制

**技术复杂度**：低
**数据依赖**：无新数据依赖，基于现有AbstinenceRecord实体扩展

**优先级**：P0

### Feature 02：渴望应对面板2.0

**产品经理描述**

这是清流的核心差异点。用户说"最需要的是在渴望来袭时3秒内打开一个能真正起作用的东西"。我们要做的不是展示11个技术的列表，而是告诉用户"此刻，你最应该做哪个"，而且这个推荐要有神经科学的依据，不是经验主义。

**功能描述**

#### 2.1 技术优先级重排

基于tech_mechanism.md研究，前端展示优先级调整：

| 技术 | 优先级 | 位置 |
|------|--------|------|
| T11 冷水洗脸+自我对话 | 最优先（首位推荐） | 第1位 |
| T4 90秒呼吸法 | 高优先 | 第2位 |
| T5 感官接地54321 | 高优先 | 第3位 |
| 其他技术 | 保留 | 可折叠区域 |

> 注：T2"肾上腺素-多巴胺拮抗"描述修正为"急性运动→前额叶激活→抑制性控制增强"；T10强震感换场从"增强"降级为"实验性保留（无RCT证据）"

#### 2.2 T4呼吸引导动画

- 视觉引导：圆形/气泡呼吸引导（吸气扩张→屏息保持→呼气收缩）
- 时长：6轮，约90秒
- 结束语："渴望高峰通常持续90秒，你刚刚度过了一个完整周期"
- 可选：结束提示心率下降效果

#### 2.3 T5感官接地引导

- 分步骤界面：每步一个页面（5→4→3→2→1）
- 可选：语音辅助模式（说出即可，不需要打字）
- 视觉反馈：完成一步打一个勾

#### 2.4 T11两阶段引导

- 阶段1：物理干预（冷水洗脸计时，30秒）
- 阶段2：认知对话（提供标准化自我对话模板，用户默念/朗读均可）
- 对话模板：
  - "这只是神经习惯的回响，不是我的本质"
  - "90秒后这个渴望会自然衰减"
  - "我选择给前额叶一个机会"

#### 2.5 面板入口优化

- 主屏幕：独立快捷入口（图标+文字"渴望应对"）
- 推送通知："渴望来袭？3秒打开清流"（触发时机：用户历史高风险时段）
- App启动时：从任意页面<3秒可达

**验收标准（AC）**

- [ ] 渴望面板从主屏幕<3秒触达
- [ ] T4动画流畅，呼气时视觉与计时同步
- [ ] T5每步可语音输入也可手动输入
- [ ] T11冷水阶段计时准确（30秒±2秒）
- [ ] T11自我对话模板显示完整，无截断

**技术复杂度**：中（T4/T5有动画工程量）
**数据依赖**：无新数据依赖，基于现有UrgeTechnique实体扩展

**优先级**：P0

### Feature 03：破戒后支持系统

**产品经理描述**

这是行业最大的空白。所有竞品都是破戒后简单重置，而这是用户最脆弱的时刻——也是产品最需要做对事的时刻。我们要让"破戒"从一个羞耻事件变成一个学习事件。核心原则：破戒≠失败。

**功能描述**

#### 3.1 破戒识别与路由

检测到用户记录破戒时，拦截默认重置流程，显示"破戒支持引导页"（不再直接回到0天界面）。

#### 3.2 破戒分析问卷（<2分钟）

问题设计（5个问题）：

1. "这次破戒发生在什么情境下？"（地点/时间/场景选择）
2. "破戒前你的情绪状态是什么？"（多选标签：焦虑/抑郁/孤独/无聊/愤怒/压力/其他）
3. "破戒前你尝试了哪些应对方式？"（多选：深呼吸/冷水/运动/54321/其他/未应对）
4. "这次破戒中你学到了什么？"（自由文本，最少10字）
5. "下一次遇到类似情境，你会怎么做不同？"（自由文本，最少10字）

#### 3.3 科学叙事呈现

问卷完成后，显示科学叙事（非评判性）：

> "这是神经习惯的回响，不是你的失败。
> 你的前额叶正在学习一个新的反应方式，这需要时间和重复。
> 你仍保有[X]天的累计恢复时间和[X]次成功应对的经验。"

#### 3.4 数据保留规则

| 数据类型 | 处理方式 |
|---------|---------|
| 累计戒断天数 | 保留 |
| 已学习的触发情境 | 保留 |
| 成功应对记录 | 保留 |
| 技能徽章/成就 | 保留 |
| 当前连续天数 | 清零 |
| 里程碑进度 | 清零（重新计算） |

#### 3.5 破戒后微行动推荐

问卷最后，推荐1-2个具体微步骤（从T4/T5/T11中选，基于用户刚才的情境匹配）：

- 示例："做一次90秒呼吸法，感受副交感神经激活"
- 示例："记录这次触发，为下次做准备"

**验收标准（AC）**

- [ ] 破戒记录触发后不直接进入重置页面，显示支持引导
- [ ] 问卷5个问题全部可答，阻止跳过（需完成才能继续）
- [ ] 破戒后"累计天数"显示正确，不归零
- [ ] 科学叙事呈现完整，无评判性语言
- [ ] 微行动推荐与用户刚才选择的情境匹配

**技术复杂度**：中（新增破戒引导页面 + 数据结构扩展）
**数据依赖**：基于现有abstinence_record扩展

**优先级**：P0

## 三、V1.1 Feature清单

### Feature 04：每日双端干预

**产品经理描述**

建立用户每天使用清流的习惯，不是靠推送提醒，而是靠一个真正有用的早晨和睡前仪式。参考I Am Sober的行业最佳实践，但要更简洁，3分钟内完成。

**功能描述**

#### 4.1 早晨承诺（打开App <10秒）

显示内容：
- 今日戒断宣言（一句话，用户可自定义或使用默认模板）
- 昨日简短回顾（"昨天你成功应对了3次渴望"）
- 可选：设定当日具体目标（从T4/T5/T11选一个作为今日重点）

#### 4.2 夜间回顾（睡前2-3分钟）

显示内容：
- "今天我应对了哪些挑战？"（多选：T4/T5/T11/其他）
- "今天我的状态如何？"（情绪自评1-5）
- "今天我为恢复做了什么？"（自由文本，最少5字）
- 简短统计（连续天数、本周应对次数）

#### 4.3 推送通知

| 推送类型 | 时间 | 内容 |
|---------|------|------|
| 早晨提醒 | 用户设定（如8:00） | "新的一天开始了，今天你有什么承诺？" |
| 夜间提醒 | 用户设定（如22:00） | "睡前花2分钟回顾今天" |
| 高风险提醒 | 基于历史数据动态触发 | "今晚孤独感强？记得你的应对工具在这里" |

**验收标准（AC）**

- [ ] 早晨承诺<10秒完成（不含跳过动画）
- [ ] 夜间回顾不超过3分钟
- [ ] 推送通知用户可关闭，可自定义时间
- [ ] 早晨/夜间数据与进度报告打通

**技术复杂度**：低-中（推送通知系统 + 简洁UI）
**优先级**：P1

### Feature 05：触发日志增强版

**产品经理描述**

触发日志是Marlatt复发预防模型的数字化实现。我们要的不只是"记录发生了什么"，而是让用户看到自己的模式，然后主动预防。

**功能描述**

#### 5.1 记录字段（扩展）

| 字段 | 类型 | 说明 |
|------|------|------|
| 时间戳 | 自动 | 精确到分钟 |
| 触发场景 | 选择+自由文本 | 地点/时间/前置事件 |
| 情绪状态 | 多选标签 | 焦虑/抑郁/孤独/无聊/愤怒/压力/其他 |
| 渴望强度 | 1-10量表 | 用户自评 |
| 应对方式 | 多选 | T4/T5/T11/其他/未应对 |
| 结果 | 选择 | 成功抵抗/破戒/部分成功 |
| 备注 | 自由文本 | 可选 |

#### 5.2 情绪-触发关联分析

- 情绪热力图：展示哪种情绪最常触发渴望（周视图）
- 高风险情境清单：基于历史数据生成个人化清单
- 主动推送：当用户记录负面情绪时，提示"你在这个情绪下通常需要应对工具"

**验收标准（AC）**

- [ ] 记录流程<1分钟完成
- [ ] 情绪标签可多选
- [ ] 热力图周视图可查看
- [ ] 负面情绪记录后30分钟内推送提示（可关闭）

**技术复杂度**：中（数据可视化 + 主动推送逻辑）
**优先级**：P1

---

### Feature 06：睡眠质量追踪

**产品经理描述**

睡眠和自制力之间的关系是被神经科学验证的。我们要做的不是监测用户睡眠（这不现实且侵犯隐私），而是让用户每天自评，同时告诉他们"睡眠不足会削弱你的前额叶功能"。

**功能描述**

- 每日睡眠自评（1-5分量表，睡前或晨起）
- 睡眠-自制力关联提示：当连续3天睡眠评分<3时，显示"睡眠不足会削弱前额叶功能——这可能是渴望来袭的重要原因"
- 睡眠趋势图（周视图，与触发日志关联）

**验收标准（AC）**

- [ ] 睡眠评分每日可记录一次
- [ ] 连续3天低分触发提示（可关闭）
- [ ] 睡眠趋势图与触发热力图可并列查看

**技术复杂度**：低
**优先级**：P1

## 四、V1.2 Feature清单

### Feature 07：AI即时对话支持

**产品经理描述**

这是竞品差异化的核心机会。用户说"渴望来的时候，最想要的是有人能即时回应"。不是冰冷的机器人，是真的理解你在什么情境下、什么情绪里、什么渴望强度，然后给出个性化的建议。

**功能描述**

- 渴望来袭时，用户可选择"AI对话"进入即时对话
- AI基于用户当前情绪、情境、历史上成功的应对方式，推荐具体技术
- 对话内容本地处理或匿名化（严格隐私保护）
- AI回复需经过临床专家审核（产品上线前）

**验收标准（AC）**

- [ ] AI响应<5秒（无需等待）
- [ ] AI推荐与用户情境匹配（非通用回复）
- [ ] 对话内容严格隐私保护，有清晰的隐私说明
- [ ] AI对话内容有"本产品不替代专业心理咨询"的提示

**技术复杂度**：高（AI对话系统 + 临床审核流程）
**优先级**：P2（需要更多资源）

---

### Feature 08：匿名社会联结（社区）

**产品经理描述**

社会联结让成瘾易感性个体风险降低52%。但我们不做NoFap那样的公开大社区——那里面有毒性和表演性。我们要的是一对一的、匿名的、真正在乎对方成功的人。

**功能描述**

- MVP版：匿名一对一伙伴匹配（互相发送简短鼓励，非实时聊天）
- 周报分享：用户可选择将周报摘要发送给伙伴
- 社区论坛（V1.2后期）：严格审核机制，禁止破戒羞辱内容

**验收标准（AC）**

- [ ] 伙伴匹配基于成瘾类型和时区，不暴露真实身份
- [ ] 鼓励消息有字数限制（不超过200字），防止骚扰
- [ ] 社区论坛有内容审核（AI预审+人工复核）

**技术复杂度**：高（匹配算法 + 内容审核）
**优先级**：P2

---

### Feature 09：效果追踪与数据报告

**产品经理描述**

用户需要看到自己的进步——数据化的、可视化的进步。这是增强内在动机的核心工具，也为我们未来做临床研究积累真实世界数据。

**功能描述**

- 个人效果数据面板（周/月视图）
- 自动生成个人化效果周报（可分享）
- 为未来临床研究预留数据收集接口（需用户明确同意）

**验收标准（AC）**

- [ ] 效果面板展示用户核心指标趋势
- [ ] 周报自动生成，可选择发送给伙伴
- [ ] 数据收集有明确的用户同意流程（隐私合规）

**技术复杂度**：中（数据可视化 + 报告生成）
**优先级**：P2

## 六、用户旅程地图（MVP）

```
用户首次打开App
    ↓
设置戒断目标（开始计时）
    ↓
每日使用
    ├── 早晨：承诺
    ├── 日常：应对渴望（T4/T5/T11）
    └── 夜间：回顾 + 记录触发
    ↓
（破戒发生）
    ↓
破戒支持引导（问卷 + 科学叙事）
    ↓
保留累计数据，重新开始
    ↓
里程碑达成（7/30/90天）
    ↓
持续使用 → 留存率提升
```
