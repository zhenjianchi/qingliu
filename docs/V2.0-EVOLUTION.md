## 1. 起点：问题与诊断

### 1.1 用户反馈

**症状**: 市场反应不好

通过提问定位：
| 问题 | 选择 |
|------|------|
| 表现是什么？ | **下载量/使用率低** |
| 目标用户？ | **年轻人（18-30）戒色/脱瘾** |
| 用户流失环节？ | **功能单薄、界面交互不好看** |

### 1.2 现有项目评估

**已有能力**（在 `Features.md` / `PRD.md` / `Research/` 中定义）：
- 戒断计时器（基于 BLoC）
- 11 种循证应对技术
- 渴望应对面板（智能推荐）
- 破戒支持系统
- 触发日志
- 神经恢复科学叙事
- 6 语言国际化

**核心短板**：
- ❌ UI 走医疗蓝（`#0077B6`），缺乏年轻用户的活力感
- ❌ 没有成就/游戏化体系，缺乏持续动力
- ❌ 首次启动是空状态，价值感知不足
- ❌ 进度展示抽象，看不见自己的"成长"
- ❌ 功能散落在多个页面，缺乏统一的"仪式感"

## 2. 决策：方向与方案

### 2.1 选定方向

经过三选一 + 用户偏好筛选：

| 方案 | 决定 |
|------|------|
| 社区 + UI 大改版 | ❌ 不做社区（用户偏好） |
| AI 陪伴 + 游戏化 | ❌ 暂缓（成本） |
| **大改版 UI + 丰富功能** | ✅ **选定** |

**具体目标**：
- UI 大改版（告别医疗蓝 → 游戏化色彩）
- 增加游戏化系统（XP / Level / 成就徽章）
- 增加情绪追踪（DayWall）
- 增加 Onboarding（解决首次启动问题）
- 增加进度可视化（让"进步看得见"）

### 2.2 参考设计系统

调研 Open Design ([open-design.ai](https://open-design.ai)) 后选定 **Lingo** 系统：

| 维度 | Lingo Token | 用途 |
|------|-------------|------|
| Primary | `#58CC02` | Duolingo 绿 - 进步、活力 |
| Secondary | `#CE82FF` | 薰衣草紫 - 平静、专注 |
| Warning | `#FFC800` | 暖黄 - 能量 |
| Danger | `#FF4B4B` | 警示红 - 紧迫 |
| Font | Nunito | 圆润、友好、年轻 |
| Style | Tactile 3D borders | Duolingo 厚边手感 |

**拒绝项**：
- ❌ 圆形进度环（用户反馈"不好看、不现代"）
- ❌ 蓝灰医疗色调
- ❌ 系统字体

## 7. 关键设计决策记录

### 7.1 为什么用 Lingo 而非 Material 默认？

| 维度 | Material | Lingo |
|------|----------|-------|
| 主色 | 紫/蓝（沉稳） | 绿/紫（活力） |
| 字体 | Roboto | Nunito（圆润） |
| 边框 | 1px 浅灰 | 2-3px 厚边 + 阴影 |
| 风格 | 系统化 | 游戏化 |
| 年轻用户匹配 | 中 | **高** |

### 7.2 为什么抛弃圆形进度环？

用户明确反馈："不好看、不现代"。
替代方案：**大数字 + 渐变背景**（Hero）+ **StepCounter 组件**

### 7.3 为什么先做 Onboarding？

诊断结论：用户首日流失率高 = 下载量低。
Onboarding 是**唯一**能在不增加获客成本的前提下提升激活率的产品杠杆。

### 7.4 为什么 XP/Level 而非成就优先？

- XP/Level 是**持续动力**（每次操作都+XP）
- 成就徽章是**里程碑奖励**（离散事件）
- 两者结合 = 持续+爆发的双重激励
